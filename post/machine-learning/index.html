使用梯度下降算法的小技巧

1.特征缩放 Feature scaling
对于特征值的区间相差很大的，其代价函数画出的图会很狭长(对于两个特征值，其图是狭长的椭圆)，这样找到的最优值的路径十分曲折，耗时。可以把特征值区间降到0到1(特征缩放)， 这样画出来的图比较圆滑，找的路径也比较快捷，梯度下降算法也能更快的收敛。
2.归一化 Mean normalization
即在特征缩放之前，先用特征值Xi减去平均值ui。

对于α学习率的值的大小的选择来说，
太小，则收敛慢(slow convergence)
太大，则随着迭代次数的增加，J(θ)可能不会下降(波动或上升)，并且有可能不会收敛(may not convergence)

那么如何选择合适的学习率？

问题来了，我们知道pinv和inv操作是求矩阵的逆，如果所求矩阵不可逆怎么办？

出现这种问题有两个原因：
1.所选的feature之间有线性关系，比如x1和x2都是代表房子的面积，只不过x1的单位是平方英尺，x2的单位是平方米，我们知道1米=3.28英尺
2.选的feature太多了
实际上，所求的矩阵不可逆的情况很少出现，所以基本不用担心！

function J = costFunctionJ(x,y,theta)//传入的参数分别为x是构造矩阵，e.g.: x=[1 1;1 2;1 3]
// y是向量，y=[1;2;3] // theta是参数向量，theta=[0;1]

sqrErrors = (predictions-y).^2;// 求方差，进行的是位运算，对两个向量的差，求其平方
J = 1/(2*m) * sum(sqrErrors);//J为cost函数的损耗值
